Creating a Visa Direct Transaction

Overview

Our Visa Direct API allows a merchant to send funds directly to a customer’s Visa branded card. Now you can pay your customers directly on their Visa cards.

You will first have to create a Visa Direct Transaction with the
https://pp.payper.ca/api/v2/payout/VisaDirect/CreateTransaction endpoint.

📘

Refer to Visa Direct for more information on this endpoint.

Request

{
     "amount": 37,
     "first_name": "Brandon",
     "last_name": "Martin",
     "email": "Brandon@Martin.com",
     "phone": "+14034881071",
     "address": "3291 Craven Place",
     "city": "Medicine Hat",
     "state": "Alberta",
     "country": "CA",
     "zip_code": "T1A 0N1",
     "ip_address": "127.0.0.1",
     "ntf_url": "https://notification.url/",
     "udfs": [
          "123456"
     ]
}

Response

{
  "amount": 1,
  "descriptor": "VisaDirectMoneySend",
  "message": "Visa Direct payout transaction created",
  "status": "approved",
  "tx_action": "request",
  "txid": "30055317283286874987777889094",
  "udfs": [
    "udf1",
    "udf2"
  ],
  "cards": [
    {
      "name": "John Glen",
      "card_id": 1234,
      "last_4_digits": 4005
    },
    {
      "name": "John Glen",
      "card_id": 1234,
      "last_4_digits": 1002
    }
  ]
}

Postbacks

When Payper receives a request, Payper will send postbacks to the notification endpoint you provide for ntf_url. A postback indicates that a change has been made to the object referenced in the event, such as changes to the transaction action and status. For example, a request could be pending but as soon as it changes to approved, a postback will be sent notifying that this change has occured. For more information on how to set up postbacks, refer to Conditions for sending webhooks.

First Postback

Here's an example of the first postback. This is sent when the status request changes from pending to approved. Refer to Postback Notifications for more information.

{
    "txn": "30230957487534930094543158744",
    "stage": 1,
    "tx_action": "request",
    "status": "approved",
    "amount": "$41.59 CAD",
    "amount_raw": 41.59,
    "descriptor": "CreditSendVisa",
    "udfs": [
        "udf1565ffc6e-baf8-49ca-818a-0b5a33773241",
        "udf2"
    ],
    "ntf_url": "https://notification.url/",
    "code": 2001
}

Second Postback

Here's an example of the second postback. This is sent when the status of payout changes from pending to approved. Refer to Postback Notifications for more information. 

{
    "txn": "30230957487534930094543158744",
    "stage": 2,
    "tx_action": "payout",
    "status": "approved",
    "amount": "$41.59 CAD",
    "amount_raw": 41.59,
    "descriptor": "CreditSendVisa",
    "udfs": [
        "udf1565ffc6e-baf8-49ca-818a-0b5a33773241",
        "udf2"
    ],
    "ntf_url": "https://notification.url/",
    "code": 5101
}
