e-Transfer Standard

post https://pp.payper.ca/api/v2/payment/eTransferStandard/

📘

Out of Band Transactions

Out of Band Transaction must be handled while using this endpoint. Please visit this page for more information: https://docs.payper.ca/reference/out-of-band-postback

Our API will return a JSON response to your system. The following is a description of the response fields that you should expect.

ParameterTypeNoteDescription
amountDECIMALThe amount that was processed.
bank_payment_urlSTRINGIf bank_id is provided in the request, this field will contain a link to the bank for the customer to complete the request money transaction. If bank_id is not provided this field will contain a list to the Interac bank selector.
descriptorSTRINGContains the descriptor used that will be displayed on the customer's bank statement.
errorsARRAYContains a list of any errors associated with the request.
messageSTRINGContains the instructions necessary to complete the transaction. During an error/declined state this field will contain the reason for the decline.
payment_qrcodeSTRINGA QR code containing the 'bank_payment_url'
standard_emailSTRINGThe email address to display to the customer to send the eTransfer to.
standard_messageSTRINGThe message to display to the customer that should be provided when sending the eTransfer.
standard_secret_answerSTRINGThe secret answer to display to the customer that should be provided when sending the eTransfer.
statusSTRINGDetermines the outcome of the transaction.
NOTE: An approved status does not mean the payment been processed, only that the request has been approved.
tx_actionSTRINGThe current action performed.
redirectBOOLEANIf value is set to true the customer should be redirected to the bank_payment_url. If value is set to false a pop-up should be opened to the bank_payment_url
txidSTRINGOur internal 29 digit transaction identifier.
udfsARRAYUser defined fields as a STRING array. One field is required and must be unique to every transaction. This field is returned unmodified in the response.
convenience_feeFLOATThe 'convenience_fee' given in the request
