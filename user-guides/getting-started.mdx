### 3. API Communication

<Note>
Communication with the PayPer API is conducted by passing JSON strings via HTTPS POST to our secure endpoints.
</Note>

**Base URL**: `https://pp.payper.ca/api/v2/`

**Headers Required**:
```json
{
  "Content-Type": "application/json",
  "Authorization": "Bearer YOUR_API_TOKEN"
}
```

**Response Format**: All API responses return JSON with HTTP status codes `200-299` for successful operations.

## Integration Examples

### Basic Payment Flow

```mermaid
sequenceDiagram
    participant C as Customer
    participant Y as Your App
    participant P as PayPer API
    participant B as Bank
    
    C->>Y: Initiate Payment
    Y->>P: Create Payment Request
    P->>Y: Return Payment Instructions
    Y->>C: Display Payment Options
    C->>B: Complete Payment
    B->>P: Payment Confirmation
    P->>Y: Webhook Notification
    Y->>C: Success Confirmation
```

### Basic Payout Flow

```mermaid
sequenceDiagram
    participant Y as Your App
    participant P as PayPer API
    participant R as Recipient
    participant B as Bank
    
    Y->>P: Create Payout Request
    P->>R: Send Payment Notification
    R->>B: Accept/Deposit Payment
    B->>P: Payout Confirmation
    P->>Y: Webhook Notification
```
