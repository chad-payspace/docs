## Overview

Digital Cheque (EFT) enables customers to pay directly from their bank accounts with lower fees and reduced chargebacks compared to credit cards.

**Key Benefits:** Lower processing fees ‚Ä¢ Direct bank transfer ‚Ä¢ Reduced chargebacks ‚Ä¢ No credit card required

## Quick Start

Digital Cheque processing is simple: customer provides bank details ‚Üí payment authorization ‚Üí funds transfer ‚Üí confirmation.

## API Integration

**Endpoint:** `POST https://api.payper.ca/v1/digital-cheque/transaction`  
**Authentication:** Bearer token in Authorization header

### Required Parameters
- `amount` - Transaction amount
- `first_name`, `last_name` - Customer's name
- `email` - Customer's email  
- `phone` - Customer's phone number

### Optional Parameters
- `address`, `city`, `state`, `country`, `postal_code` - Customer address

## Response Format

**Success Response:**
```json
{
  "txid": "dc_1234567890",
  "status": "APPROVED",
  "message": "Transaction approved", 
  "amount": 150.00
}
```

**Response Status:**
- `APPROVED` - Transaction successful
- `DECLINED` - Transaction declined  
- `PENDING` - Processing in progress

## Error Handling

**Common Errors:**
- `400` - Invalid parameters
- `401` - Invalid API key
- `422` - Validation failed

```json
{
  "error": {
    "code": "validation_failed",
    "message": "Invalid email format"
  }
}
```

## Security Best Practices

<Warning>
  Never store sensitive banking information. Always use HTTPS and secure API key storage.
</Warning>

**Key Requirements:**
- Store API keys securely (server-side only)
- Use HTTPS for all requests
- Validate input data
- Follow PCI compliance guidelines
- Obtain customer consent for EFT processing
